Documentation

Input Format

Use the table to add processes with the following parameters:

  • ID: Process identifier (P1, P2, etc.)
  • Arrival Time: When process enters ready queue
  • Burst Time: How long process needs CPU
  • Priority: Used only in Priority Scheduling (lower = higher priority)

Algorithms Explained

FCFS - First Come First Serve

Non-preemptive algorithm where processes execute in order of arrival. Simple but can cause convoy effect.

Best for: Batch systems, simple scheduling

SJF - Shortest Job First

Non-preemptive algorithm that selects the process with smallest burst time. Minimizes average waiting time.

Best for: Minimizing average wait time

Priority Scheduling

Non-preemptive algorithm where processes are assigned priorities. Lower number = higher priority.

Best for: Real-time systems, OS kernel processes

Round Robin

Preemptive algorithm where each process gets a fixed time quantum. Fair to all processes.

Best for: Time-sharing systems, responsive systems

Formulas

  • Completion Time (CT): Time when process finishes
  • Turnaround Time (TAT): CT - Arrival Time
  • Waiting Time (WT): TAT - Burst Time
  • CPU Utilization: (Total Busy Time / Total Time) × 100%
